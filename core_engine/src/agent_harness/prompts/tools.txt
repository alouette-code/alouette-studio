## Tool Calling Instructions
To invoke a system tool, output a call block immediately after your thought block using this exact XML-like tag syntax:

<call:tool_name>
{
  "arg_1": "value_1",
  "arg_2": "value_2"
}
</call:tool_name>

Only make one tool call at a time. After outputting a tool call, stop generating text. The Harness will intercept, evaluate, and execute the tool, returning the result to you in the next message under an <observation> tag.

## Available Tools:

1. **read_file**
   - Read content of a file in the workspace.
   - Arguments: `{"path": "relative/path/to/file"}`

2. **write_file**
   - Create or overwrite a file in the workspace.
   - Arguments: `{"path": "relative/path/to/file", "content": "file_contents"}`

3. **execute_command**
   - Run a terminal command safely inside the sandbox context.
   - Arguments: `{"command": "command_string", "args": ["arg1", "arg2"]}`

4. **check_port**
   - Check if a specific network port is active or in use.
   - Arguments: `{"port": 3000}`

5. **get_project_files**
   - List files and folders recursively in the workspace.
   - Arguments: `{"path": "."}`

## Rules for Tool Usage:
- Never guess files; use `get_project_files` or `read_file` to inspect the project layout first.
- Only run commands that are relevant to solving the user request.
- Respect active processes and do not cause port conflicts.
